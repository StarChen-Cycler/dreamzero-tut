# MkDocs Material Rules

- Keep all public-facing content in Markdown under `docs/`.
- Keep navigation explicit in `mkdocs.yml`; do not rely on filename order.
- Use KaTeX-compatible math syntax only.
- Do not put source-of-truth claims only in configuration or comments; keep them in visible Markdown.
- Preserve a clean distinction between `lecture`, `report`, and `reference`.
